Fluxo recomendado

Instale o add-in, abra o Revit e suba o MCP pelo launcher.

O add-in roda dentro do Revit e recebe comandos do servidor local. O launcher deixa o fluxo visivel para quem vai usar: iniciar servidor, publicar URL quando necessario e validar a conexao.

1

Instale

Execute RevitMcpSetup.exe publicado no GitHub Releases.

2

Abra o Revit

Reinicie o Revit depois da instalacao para carregar o arquivo .addin.

3

Inicie o MCP

No launcher, clique em Start MCP e use a URL http://127.0.0.1:8000/mcp.

4

Valide

Chame get_revit_context e confirme que o projeto ativo aparece na resposta.

Uso ideal

Feito para automacao real no modelo Revit.

Contexto do projeto

Leia versao do Revit, documento ativo, caminho do arquivo e estado basico da conexao.

Consulta de elementos

Liste niveis, vistas, paredes, tipos, familias, parametros e quantidades do modelo.

Execucao por ExternalEvent

Rode snippets C# no processo do Revit, com transacoes quando a automacao altera o modelo.

Exemplo pratico

A IA pode transformar um pedido em uma acao no Revit.

O usuario pede em linguagem natural. O cliente MCP envia um snippet C# para o add-in, o Revit executa e devolve o resultado.

run_revit_code 
using (var tx = new Transaction(doc, "Criar parede"))
{
    tx.Start();
    // Revit API: cria ou altera elementos aqui.
    tx.Commit();
}

return "Automacao concluida.";

Limites atuais

O fluxo precisa do Revit aberto e do add-in carregado.

  • O servidor MCP local precisa estar rodando antes das chamadas do cliente MCP.
  • Operacoes que modificam o modelo devem usar Transaction.
  • Snippets devem ser pequenos, objetivos e revisaveis antes de executar alteracoes amplas.
  • Use tunnel apenas quando o cliente MCP exigir uma URL publica.
