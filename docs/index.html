Fluxo recomendado

Instale o add-in, abra o Revit e suba o MCP pelo próprio Ribbon.

O add-in roda dentro do Revit e recebe comandos do servidor local. O painel Revit MCP no Ribbon deixa o fluxo visível para quem vai usar: iniciar servidor, copiar URL e validar a conexão.

1

Instale

Execute RevitMcpSetup.exe publicado no GitHub Releases.

2

Abra o Revit

Reinicie o Revit depois da instalação para carregar o arquivo .addin.

3

Inicie o MCP

No Ribbon Revit MCP, clique em Start MCP e use a URL http://127.0.0.1:8000/mcp.

4

Valide

Chame get_revit_context e confirme que o projeto ativo aparece na resposta.

Uso ideal

Feito para automação real no modelo Revit.

Contexto do projeto

Leia versão do Revit, documento ativo, caminho do arquivo e estado básico da conexão.

Consulta de elementos

Liste níveis, vistas, paredes, tipos, famílias, parâmetros e quantidades do modelo.

Execução por ExternalEvent

Rode snippets C# no processo do Revit, com transações quando a automação altera o modelo.

Exemplo prático

A IA pode transformar um pedido em uma ação no Revit.

O usuário pede em linguagem natural. O cliente MCP envia um snippet C# para o add-in, o Revit executa e devolve o resultado.

run_revit_code 
using (var tx = new Transaction(doc, "Criar parede"))
{
    tx.Start();
    // Revit API: cria ou altera elementos aqui.
    tx.Commit();
}

return "Automação concluída.";

Limites atuais

O fluxo precisa do Revit aberto e do add-in carregado.

  • O servidor MCP local precisa estar rodando antes das chamadas do cliente MCP.
  • Operações que modificam o modelo devem usar Transaction.
  • Snippets devem ser pequenos, objetivos e revisáveis antes de executar alterações amplas.
  • Use URL pública fixa apenas depois de configurar ngrok e autenticação no painel de settings.
